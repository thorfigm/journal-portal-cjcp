# journal-portal-cjcp

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。页面内容主要涉及信息展示与索引功能，不针对任何特定域名或网站。仓库以静态文档形式托管，适合直接通过 GitHub Pages 或其他静态服务进行访问。

## 目录说明

```
.
├── index.html          # 主入口页面
├── pages/              # 存放其他独立 HTML 页面
│   └── ...
├── assets/             # 资源文件（样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── ...
└── README.md           # 本文件
```

- `index.html`：仓库的主页面，通常作为访问入口。
- `pages/`：包含其他独立页面，每个页面可独立访问，彼此之间无强依赖。
- `assets/`：存放页面所需的公共资源，如样式表、脚本文件、图标等。

## 页面归档说明

本仓库中的 HTML 页面均为独立归档，每个页面可单独打开使用。页面之间可能存在链接关系，但不构成完整的 Web 应用。归档内容以静态形式保存，不依赖后端服务或数据库。

如需新增页面，请遵循以下基本规则：

- 页面文件统一存放于 `pages/` 目录下。
- 公共资源（如 CSS、JS）优先使用 `assets/` 目录下的文件。
- 页面应保持自包含，避免引用外部不可控资源。

## 维护说明

本仓库的维护工作主要包括：

- 定期检查页面是否可正常显示。
- 根据需要更新或新增归档页面。
- 保持目录结构清晰，文件命名规范。

欢迎通过 Issue 或 Pull Request 提出改进建议。提交前请确保页面内容符合通用归档要求，不包含具体域名、营销信息或诱导性内容。
